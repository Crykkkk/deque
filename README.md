# Deque README


## 项目概述

本仓库主要使用分块链表（Unrolled Linked List）数据结构实现了双端队列（Deque），支持快速随机访问与动态容量调整。

## 内容介绍

Deque 的实现没有使用 STL 库，主要依托于 [Linkedhashmap](https://github.com/Crykkkk/LinkedHashMap-LRU) 实现的 Doublelist 双向链表结构，结合内部的循环数组实现。

Deque 实例的数据由数据块组成的双向链表维护，其中每个数据块的大小为选定的超参数 $M = 256$，在理想情况下认为 $M = \Theta(\sqrt n)$。数据块的实现为循环列表，通过可变的 head 和 size 数据，可实现 $O(1)$ 的头尾插入删除与随机访问。

分块链表的基本功能与运行逻辑如下：
1. **随机查询**：对于查询位置 `idx`，在外部链表对数据块进行遍历，每次将剩余元素与目标数据块的 `size` 进行比较，判断进入数据块进行遍历或进入下一数据块
2. **随机插入**：首先判断目标插入位置是否已经达到 $M$，若已达到，则先将目标块分裂为两个块 $A, B$，并将原块 $[0 , \frac{M}{2}), [\frac{M}{2}, M)$ 位置的数据分别放入 $A, B$，再通过对外部双向链表的调整将两个块按先后顺序放入原块所在的位置，最后根据插入位置在原数据块的索引将目标块放入 $A$ 或 $B$ 的指定位置，在这一索引后的数据部分向右平移
3. **头尾插入**: 对于头插入，首先同样判断头数据块是否需要分裂，如需分裂则先实施这一步骤，然后利用循环数组的性质，直接向左调整头数据块 `head` 的值（若坐标本身为 `0` 则循环至 `M - 1`）；对于尾插入，即相当于插入到最后一个数据块的末尾，则同样判断完是否需要分裂，随后在最后一个数据块的末尾插入即可，故头与尾的最后一步插入都不需要移动元素
4. **随机删除**：首先直接删除目标位置元素，并将其后的元素向左平移。删除后进行判断，若
   1. 删除后块变空，则直接删除该块
   2. 其相邻块（先右后左）与其元素数之和不大于 $3M / 4 = 192$（此处的阈值设置是为了防止在块大小处于分裂边缘时，插入和删除导致块频繁分裂与合并），则将两个块进行合并，其中元素按先后顺序重新放入合并的块中
5. **头尾删除**：在直接删除这一部分，对头部删除直接向右调整头数据块 `head` 的值并清空原位置的值即可，对尾插入，删除元素不需要移动别的元素；随后再与随机删除类似对合并情况进行判断即可

此外，还有一些基本的功能如
1. 生成迭代器并支持迭代器的比较，移动（特别的，随机查询复用了迭代器的随机移动实现）
2. 队列的基本生命周期如创建，拷贝构造，清除，析构等


## 复杂度分析

对第二部分中的主要功能进行复杂度分析，特别的，以下的复杂度分析均基于 $M = \Theta(\sqrt n)$ 的假设，对于进一步的改进，可能可以考虑随着总数据数变化动态调整 $M$，或在用户输出时可选初始化参数 $M$：

首先我们分析场上能够存在的最大块数。对此的基础分析主要来源于两相邻块的和不大于 $3M / 4$ 时，删除任一边将导致合并，另一方面，如果相邻两个块在其中一个块诞生后就不再有删除操作，那由于块的诞生总是从分裂来，这样的相邻块之和不小于 $M / 2$，因此，我们可以得到相邻块的大小之和不小于 $M / 2$，从而得到总块数的一个（不紧）上界为 $\lceil 4N / M \rceil$，即 $O(\sqrt n)$


1. **随机查询**：从头开始一块一块遍历的计算次数由块数分析为 $O(\sqrt n)$ 次，找到目标数据块后，内部循环数组的查询是 $O(1)$ 的，故总复杂度为 $O(\sqrt n)$
2. **随机插入**：首先，最坏情况下也平均 $\Theta(\sqrt n)$ 次操作才需要一次分裂，这是因为在初始化后，出现新块当且仅当发生了插入后的分裂，而每个新块都需要 $M / 2 = \Theta(\sqrt n)$ 次插入才会需要分裂。每次分裂，对双向链表的调整（直接改指针）是 $O(1)$ 的，而构造两个新块分别需要 $O(\sqrt n)$ 次操作。因此，分裂在每次随机插入的均摊复杂度为 $O(1)$，另一方面，块内的插入后平移也至多需要 $O(\sqrt n)$ 次操作，综上，随机插入的最坏复杂度为 $O(\sqrt n)$（因为分裂保证了块内元素数量最多为 $M$），平均复杂度实际上也是 $O(\sqrt n)$
3. **头尾插入**：如内容介绍部分所言，插入后平移这一步骤在头尾插入不存在，故插入部分的复杂度为 $O(1)$，而我们知道分裂在每次随机插入的均摊复杂度为 $O(1)$，故头尾插入的总均摊复杂度为 $O(1)$
4. **随机删除**：删除元素并平移的最坏复杂度为 $O(\sqrt n)$，另外，删除空块也是一个 $O(1)$ 操作。下面考察合并：一次合并的复杂度显然是 $O(\sqrt n)$，而同随机插入，每次新块产生当且仅当发生了分裂操作，分裂后的两块新块大小均为 $M / 2$，因此至少平均需要 $[2 \times (M / 2) - 3M / 4] / 2 = M / 8$ 次删除操作才会触发一次合并，且合并是不连锁的，故合并发生的平均周期为 $\Theta(n)$，得到其均摊复杂度为 $O(1)$，综上，随机删除的最坏复杂度与平均复杂度均为 $O(\sqrt n)$
5. **头尾删除**：与头尾插入类似，删除后平移这一步骤也不存在，故删除的复杂度为 $O(1)$，合并的均摊复杂度为 $O(1)$，故头尾删除的均摊复杂度为 $O(1)$

特别的，为了方便初始化与数据的移动，数据结构内部使用指针维护数据，通过进行指针交换完成块内平移，优化了大型对象插入与删除的速度。
